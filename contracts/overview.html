Core Contractsâ€‹

OmniDragonâ€‹

The main token contract that implements the ERC-20 standard with LayerZero compatibility. Key features include:

  • Cross-chain token transfers
  • Secure minting and burning mechanisms
  • Access control for administrative functions
  • Integration with VRF providers for randomness
  • Built-in jackpot mechanics and fee structure

Key Functionsâ€‹

    uint16 _dstChainId,

) external payable;

OmniDragonVRFConsumerâ€‹

Manages the integration with verifiable randomness providers:

  • Multiple VRF source integration
  • Randomness verification
  • Request and callback mechanisms
  • Fallback randomness sources

Security Featuresâ€‹

Our contracts implement several security measures:

  • Role-based access control (RBAC)
  • Multi-signature requirements for critical operations
  • Rate limiting and circuit breakers
  • Comprehensive event logging
  • Regular security audits

Access Controlâ€‹

// Role-based access through Ownable pattern

    require(owner() == msg.sender, "Caller is not the owner");
    _;

// Authorization for consumers

Developmentâ€‹

For developers looking to contribute or build on OmniDragon:

  1. Clone our repository:

    git clone https://github.com/wenakita/omnidragon.git

  2. Install dependencies:

    npm install

  3. Run tests:

    npm test

Securityâ€‹

Security is our top priority. Our contracts have undergone multiple audits and are continuously monitored. If you discover any security issues, please report them to security@sonicreddragon.io.

Security Best Practicesâ€‹

  1. Access Control

    • Always use the provided role-based access control
    • Implement multi-signature for critical operations
    • Regularly review and update access permissions

  2. Cross-Chain Operations

    • Verify message sources
    • Implement replay protection
    • Use appropriate gas limits
    • Monitor bridge operations

  3. Randomness Usage

    • Verify randomness proofs
    • Implement fallback mechanisms
    • Use appropriate timeouts
    • Monitor VRF network status

Contract Categoriesâ€‹

The OmniDragon contracts are organized into the following categories:

Coreâ€‹

  • OmniDragon.sol: Main token contract with cross-chain capabilities
  • OmniDragonPeriphery.sol: Helper contract for managing token integrations
  • ChainRegistry.sol: Manages endpoints across supported chains
  • ChainSpecificEndpoint.sol: Chain-specific configurations

Jackpotâ€‹

  • DragonJackpotDistributor.sol: Distributes jackpot winnings to winners
  • DragonJackpotVault.sol: Securely stores jackpot funds
  • OmniDragonSwapTriggerOracle.sol: Creates lottery entries from swaps

Mathâ€‹

  • DragonMathLib.sol: Core mathematical utilities
  • HermesMath.sol: Jackpot distribution mathematics
  • DragonDateTimeLib.sol: Time-related calculations
  • ve69LPMath.sol: veToken staking mathematics
  • VotingPowerCalculator.sol: Governance voting power calculations
  • DragonAdaptiveFeeManager.sol: Dynamic fee adjustment system
  • MarketConditionOracle.sol: Market condition monitoring for adaptations

Randomnessâ€‹

  • OmniDragonVRFConsumer.sol: Main verifiable randomness consumer
  • DragonVRFIntegrator.sol: External randomness integration
  • DragonVRFConsumer.sol: Base randomness consumer contract

Governanceâ€‹

  • OmniDragonGovernor.sol: Governance contract for protocol decisions
  • OmniDragonTimelockController.sol: Timelock for governance actions
  • ve69LP.sol: Vote-escrowed LP token for governance

Partnersâ€‹

  • DragonPartnerRegistry.sol: Registry for partner integration
  • DragonPartnerFactory.sol: Factory for creating partner pools
  • DragonPartnerPool.sol: Partner-specific pools and incentives
